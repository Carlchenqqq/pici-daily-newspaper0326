## 重构【导出/打印】功能为 ZIP 离线报告包

**需求背景：** 原来的导出功能是 cloneNode 整个 DOM 生成 HTML，存在依赖报错（script 被删除但 onmouseenter 保留）和交互缺失等问题。改为生成可直接双击打开的离线 ZIP 包。

**实现方案：**
1. 新增后端路由 `GET /api/export-package`（app.py）：调用 `generate_historical_report`，返回批次全量 JSON 数据
2. 新增后端路由 `GET /export-template`（app.py）：渲染并返回 `templates/export_report.html` 模板内容
3. 新建 `templates/export_report.html`：完全独立的静态报告模板，包含完整 CSS 和 JS（ECharts 曲线图、tab 切换、hover tooltip），通过 `window.REPORT_DATA` 接收数据
4. 重构 `templates/index.html` 中 `exportCurrentView` / `exportToStaticHTML` 函数（第 4107 行）：
   - 改为 `exportToPackage()` 函数
   - 调用后端 `/api/export-package` 获取最新数据
   - 调用 `/export-template` 获取模板
   - 注入 `<script>window.REPORT_DATA = {...}</script>`
   - 使用 JSZip（新增 CDN 引入）打包成 ZIP（包含报告.html + 使用说明.txt）
   - 触发浏览器下载 ZIP 文件

**用户使用方式：** 解压 ZIP，双击 `报告.html`，在浏览器中直接查看完整交互报告（hover 曲线、tab 切换）。首次打开需联网加载 ECharts CDN。

**修改文件：**
- `app.py`：新增 2 个路由
- `templates/index.html`：引入 JSZip CDN，重构导出函数
- `templates/export_report.html`：全新创建


**问题描述：** 点击「导出/打印」按钮下载的静态HTML文件，鼠标悬停表格单元格时报 `Uncaught ReferenceError: showDailyCurveTooltip is not defined` 和 `hideDailyCurveTooltip is not defined`。

**根本原因：** `exportCurrentView` 函数在导出时正确地删除了所有 `<script>` 标签（包含这两个函数的定义），但在清理DOM事件属性时（第4180-4188行），只清除了 `onclick/onchange/oninput/onsubmit/onfocus/onblur`，**漏掉了 `onmouseenter` 和 `onmouseleave`**，导致表格单元格上的这两个属性仍然保留并尝试调用已不存在的全局函数。

**修复方案：** 在 `allElements.forEach` 循环中补充移除 `onmouseenter`、`onmouseleave`、`onmouseover`、`onmouseout` 属性，文件 `templates/index.html` 约第4180行。
